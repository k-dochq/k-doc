## 8. 시스템 구조 요약 (Figma 좌측 컬럼)

### 8.1 상담 시스템 2개로 분리되는 구조

| 구분 | 병원 개별 상담 (현재) | K-DOC 통합 상담 (신규) |
|---|---|---|
| 형태 | 병원 1:1 채팅 | 채널톡 스타일 플로팅 |
| 목적 | 특정 병원 상담, 시술 상담, 예약 진행, 컨시어지 등 | 플로팅 문의, 응대, 병원 추천, 컨시어지 안내, 운영 알림, 시리즈 상담 |
| 별칭 | "에이스 상담" | "플로팅 1차 상담" |

### 8.2 이 구조의 장점

1. **역할 분리 명확** — K-DOC = 플로팅 1차 응대 / 병원 = 실제 의료 상담
2. **CRM 흐름 추적** — 관심 → 추천 → 병원 연결 → 회원가입 유도 → Mind 병원 상담 신청/생성 → 예약 전환의 전 과정 추적 가능
3. **어드민 관리 쉬워짐**
   - K-DOC 상담팀: 초기 응대 / 추천 / 컨시어지
   - 병원 상담팀: 실제 예약 / 일정 / 시술
   - 운영 분리 가능

### 8.3 추가 개발 기능 (슬랙 본문 핵심)

1. 상담채팅에서 회원가입 전환 추적용 **UTM 링크** (→ Q20-21)
2. **웹푸시** — 사이트 이탈 후에도 브라우저 푸시 수신 (→ Q22)

### 13.3 기술 메모

- **국적 드로어**: `openDrawer<string>` API (`shared/lib/drawer/drawer-api.tsx`) + `COUNTRY_CODES` (`entities/country-code`) 재사용
- **DB 영향 없음**: `KdocChatCategory` Prisma enum 유지 (라벨만 변경)
- **phase 명 변경**: `'category'` → `'main_menu'` (constants/hook/page 모두 반영됨)
- **국적 저장값**: 변경 전 자유텍스트 → 변경 후 영어 국가명 (`COUNTRY_CODES[].name`)
- **CMS 스키마 소유권**: admin 프로젝트가 마이그레이션 소유. k-doc은 `prisma db pull`로 동기화
- **CMS 데이터 흐름**: admin CMS → DB (JSON 다국어 컬럼) → k-doc `/api/chatbot-cms?locale=` → `useKdocCmsContent` (tanstack) → UI
- **스켈레톤**: CMS 로딩 중 `KdocWelcomeSkeleton` + `KdocMenuSkeleton` 표시 (`animate-pulse`)
